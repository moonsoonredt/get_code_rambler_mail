# get_code_rambler_mail
Автоматический скрипт для подключения к почтовому ящику по протоколу POP3, скачивания последнего письма и извлечения из него 6-значного кода подтверждения. Результат выводится в консоль и автоматически копируется в буфер обмена Windows.

# 📄 Документация: `get_code` — Извлекатель кодов верификации из почты

## 🎯 Назначение
Автоматический скрипт для подключения к почтовому ящику по протоколу **POP3**, скачивания последнего письма и извлечения из него **6-значного кода подтверждения**. Результат выводится в консоль и автоматически копируется в буфер обмена Windows.

---

## ⚙️ Ключевые особенности
| Функция | Описание |
|---------|----------|
| 🔍 **Умный парсинг** | Работает с `text/plain` и `text/html`. HTML автоматически очищается от тегов, спецсимволы декодируются. |
| 🌐 **Автоопределение сервера** | Автоматически выбирает `pop.rambler.ru` или `pop.mail.ru` по домену почты. Легко расширяется. |
| 📋 **Автокопирование** | Найденный код мгновенно попадает в буфер обмена через `clip`. |
| 🛡️ **Безопасное соединение** | Использует `POP3_SSL` (порт 995), таймаут 10 сек, корректное закрытие сессии. |
| 📦 **Без зависимостей** | Работает только на стандартной библиотеке Python. Не требует `pip install`. |

---

## 📦 Требования
- **Python 3.7+**
- **Windows** (для `.bat`-обёртки и команды `clip`)
- Включённый доступ **POP3** в настройках почтового ящика
- ⚠️ **Для Rambler/Mail.ru**: требуется **«Пароль для внешних приложений»**, а не основной пароль от аккаунта.

---

## 🚀 Быстрый старт
1. Положите `get_code.py` и `get_code.bat` в **одну папку**.
2. Запустите `get_code.bat`.
3. Введите строку в формате `email:пароль` и нажмите `Enter`.
4. Скрипт подключится, найдет код и покажет его в консоли. Код уже скопирован в буфер.

### 💻 Альтернативный запуск (через консоль)
```bash
python get_code.py "tzwdbalfsh@rambler.ru:72162326j1Ohs"
```

---

## 📊 Формат вывода
| Префикс | Значение |
|---------|----------|
| `CODE: 339216` | ✅ Успех. Код найден и скопирован. |
| `EMPTY: ...` | 📭 Ящик пуст или письма не загружены. |
| `AUTH_ERROR: ...` | 🔑 Ошибка входа. Обычно из-за использования основного пароля. |
| `NOTFOUND: ...` | 🔎 Письмо есть, но 6-значный код не найден в тексте/HTML. |
| `ERROR: ...` | ⚠️ Сетевая ошибка, таймаут или проблема с парсингом. |

## 🔧 Настройка под другие почтовые сервисы
В `get_code.py` найдите блок автоопределения и добавьте нужный домен/сервер:
```python
if domain in ('mail.ru', 'inbox.ru', 'bk.ru', 'list.ru'):
    pop_server = "pop.mail.ru"
elif domain == 'yandex.ru':
    pop_server = "pop.yandex.ru"
elif domain == 'gmail.com':
    pop_server = "pop.gmail.com"
    # Примечание: Gmail требует включённый POP3 + пароль приложения (OAuth2 не поддерживается в стандартном POP3)
else:
    pop_server = "pop.rambler.ru"
```

## ⚠️ Безопасность и рекомендации
- 🔒 **Не храните пароли в коде**. Для автоматизации передавайте данные через переменные окружения:  
  `python get_code.py "%MAIL_CRED%"`
- 📧 Всегда используйте **пароли для внешних приложений**, генерируемые в настройках почты.
- 🔄 Скрипт автоматически закрывает POP3-сессию (`srv.quit()`), не оставляя открытых соединений.
- 🌐 При работе в корпоративной сети убедитесь, что порт `995` не заблокирован фаерволом или прокси.

---

## 🐛 Частые проблемы и решения
| Симптом | Причина | Решение |
|---------|---------|---------|
| `AUTH_ERROR` | Используется основной пароль от почты | Сгенерируйте «Пароль для внешних приложений» в настройках аккаунта |
| `EMPTY` | Ящик пуст или письмо ещё не пришло | Проверьте веб-интерфейс, папку «Спам», задержку доставки |
| `NOTFOUND` | Код имеет другой формат или письмо только в HTML | Временно замените `print("NOTFOUND...")` на `print(content)` для отладки |
| `timeout` / `connection refused` | Блокировка сети или неверный POP3-сервер | Проверьте настройки POP3, отключите VPN, убедитесь, что порт 995 открыт |
